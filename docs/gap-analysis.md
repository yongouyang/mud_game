## 3. 关键差距分析

### 3.1 内容量：骨架已通，深度不足
oiuv_mud 的武功/NPC/物品数据已批量提取进项目，但行为同质化严重：大量武功只有名字不同、NPC 缺少特色 AI、掉落和任务关联单薄。需要一次“质量打磨”而非简单堆量。

### 3.2 社交系统几乎空白
没有 `say/tell/shout/chat`，没有公会、交易、邮件。MUD 的强社交属性无法发挥，多人在线体验接近单机。

### 3.3 后期玩法缺失
原版的核心长期目标之一是**转世系统**（多次 reborn 叠加属性、挑战 super NPC）。当前没有 reborn，也没有高阶副本/传奇装备循环。

### 3.4 生活与世界模拟不足
食物/饥饿、天气昼夜、采集小游戏、坐骑房屋等系统能显著提升沉浸感，但当前均未实现。

### 3.5 世界连通性：已解决
39 个门派现在全部可以从主世界到达（通过荒野门派岔路），地图骨架基本完整。

## 5. 下一批增强建议（分三轮）

### 🥇 Round 1.5：内容质量打磨（建议先补这一小轮）
在数据量变大的基础上做“去假存真”，让提取的内容真正可玩。

1. **武功去重与平衡** — 清理同名/重复/明显错误的武功；为每个门派保留 3-5 个有区分行为的招牌武功；补齐缺失的绝招 perform 数据。
2. **NPC 行为与位置优化** — 避免一个房间挤满 NPC；给重要 NPC 分配合适的 roomId；确保每个门派大厅至少有掌门和 1-2 个弟子。
3. **任务链扩展** — 从 3 个示例扩展到 20-30 个，覆盖各门派/城镇/荒野；增加任务奖励梯度。
4. **状态包扩充** — 从 11 个扩展到 25+，补充更多毒、内伤、药物依赖及解药。
5. **掉落与经济校准** — 让 Boss/精英怪掉落有价值，商店/拍卖价格相对合理。

### 🥈 Round 2：社交与后期系统（中等风险、延长生命周期）
框架稳定后，加入玩家互动与长期目标。

1. **聊天系统** — `say`/`tell`/`shout`/`chat` 频道，以及门派频道。
2. **玩家交易与邮件** — `give`/`trade`/`mail` 命令。
3. **帮会/门派联盟** — 玩家创建帮会、帮会频道、帮会仓库、帮会战。
4. **PvP 竞技场** — 决斗、擂台、悬赏/bounty 系统。
5. **高级任务系统** — 随机日常任务、任务等级奖励（30/50/100 次 milestone）、转世专属任务。
6. **转世 (Reborn) 系统** — 玄黄丹、多次转世属性加成、super NPC 挑战。
7. **传奇装备/副本** — 多人 Boss、副本房间、装备强化。

### 🥉 Round 3：世界模拟与生活玩法（高风险/低优先级、提升沉浸）
作为长期“调味”系统，在前两轮稳定后再投入。

1. **饥饿与食物** — 食物消耗、烹饪、酒楼。
2. **采集玩法** — 钓鱼、挖矿、采药、捕虫。
3. **小游戏** — 赌博、围棋、象棋、掷骰子。
4. **坐骑系统** — 马匹购买/喂养/加速移动。
5. **天气与昼夜** — 影响 NPC 刷新、状态、视野。
6. **玩家房屋** — 建房、储物、装修。
7. **可读书籍** — 秘籍研读、自动涨经验/技能。
8. **婚姻系统** — 结拜/结婚/离婚、夫妻技能。

## 6. 推荐下一步

当前项目已经从“框架 + 示例”进入了“有内容的世界”阶段。继续推进的话，**建议先做一次 Round 1.5 的内容质量打磨**，因为单纯扩大数据量带来的体验收益已经递减，而清理/校准/补任务的成本相对较低、效果立竿见影。

如果更想增加系统深度，也可以直接启动 **Round 2 的聊天系统**——它是后续所有社交玩法（帮会、PvP、交易）的基础设施，风险可控且能立刻改善多人在线体验。

 whichever you pick, I'll follow the same workflow: plan → implement → tests → commit/push.
